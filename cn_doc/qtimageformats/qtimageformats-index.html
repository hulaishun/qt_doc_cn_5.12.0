Qt 图片格式

核心的Qt Gui库，默认已支持大多数通用图片文件格式的读写: PNG, JPEG, BMP, GIF 和其他格式。可参考读写图片文件. Qt图片格式附加模块提供了其他图片文件格式的可选支持。

通过适用于Qt图片IO系统的插件，对支持的文件格式提供易用支持。注意，该模块对自身未提供支持。但是，可在Qt其他图片IO中，通过相同方式的访问函数基础，通过QImage::load() 和 QImage::save(). 若想了解更多详细控制，可通过 QImageReader类 和 QImageWriter类.

图片IO插件

The actual coding and decoding of the file format is done by a codec library. The codec can be Qt or third party code. In case of a third party codec, the build process will look for it among the system libraries. If not found, it may fall back on using a bundled copy (in src/3rdparty).

格式描述支持3rd Party Codec
ICNSApple Icon Image读写No
JP2Joint Photographic Experts Group 2000读写Yes (Not bundled)
MNGMultiple-image Network Graphics只读Yes (Not bundled)
TGATruevision Graphics Adapter只读No
TIFFTagged Image File Format读写Yes (bundled)
WBMPWireless Bitmap读写No
WEBPWebP读写Yes (bundled)

注意: Some bundled third party codecs are not maintained anymore. They are provided for manual builds only, and not used as a fallback in case system libraries are unavailable.

Note: For security reasons, the Direct Draw Surface (DDS) handler is not built by default since Qt 5.8. Users who still want this handler can build it from the source project.

部署

当编译打包时，Qt 图片格式插件作为动态库，位于运行时插件目录下 (典型形式 plugins/imageformats), 和默认图片格式插件一起。 这些和其他插件一样，可已同样的方式，部署到目标系统。参见 部署插件 文档。

安全事项

Since these file formats are more rarely used, the codecs may be less thoroughly debugged against potential security holes. As always, care should be taken when creating applications that may be used to decode uncontrolled data files.
